### redis底层实现

#### 1. string类型的底层实现是啥？(2_2023_11_27)
它的底层实现是sds,简单动态字符串

#### 2. sds核心结构体包含哪些字段？(2_2023_11_27)
```c++
len: 记录buf数组已使用的长度
free: buf数组中未使用的长度
buf[]: 保存字符串
```

#### 3. 为什么使用sds而不使用c自带的字符串？(2_2023_11_27)
1. o(1)时间获取字符串长度
2. 二进制安全
3. 杜绝缓冲区溢出
4. 减少修改字符串带来的内存重分配次数，惰性删除和空间预分配机制
5. 兼容部分c语言函数，已'\0'为结尾

#### 4. redis hash表的核心数据结构？(2_2023_11_27)
```c++
dictEntry table 哈希表数组
long size 哈希表大小
long sizemask 哈希表大小掩码，用于计算索引值
long used 节点数量
```

#### 5. redis字典结构包含哪些核心字段？(2_2023_11_27)
```c++
dictht ht[2];  两个hash表
int rehashidx;  rehash index，不进行rehash时为-1
```

#### 6. redis rehash可以缩容嘛？(2_2023_11_27)
可以，容量缩到第一个>= used的2^n次方

#### 7. redis扩缩容的条件？ (2_2023_11_27)
没有执行rdb或者aof，装载因子大于1的时候，执行rdb或者aof，装载因子大于5的时候，执行扩容
装载因子小于0.1的时候只想缩容

#### 8. redis也是渐进式hash嘛？ (2_2023_11_27)
是的

#### 9. 跳跃表接点包含哪些字段？ (2_2023_11_27)
```c++
level[].span 每层的跨度
level[].forward 前进指针

backward 后退指针
score 分数
obj 对象
```

#### 10. 跳跃表的时间复杂度？(2_2023_11_27)
最好O(logn),最差o(n),平均也是o(logn)的时间复杂度

#### 11. 什么时候会使用整数集合？(2_2023_11_27)
整数集合是集合的底层实现之一，当set元素不多，并且都是数字的时候，会使用整数集合

#### 12. 整数集合结构体包含哪些主要字段？(2_2023_11_27)
```c++
encoding 编码
length 长度
content[] 保存元素数组
```

#### 13. 为什么要使用整数结合？ (2_2023_11_27)
节省内存，可以是否二分查找去查找数据在整数集合中是否存在，提升效率


#### 14. 整数集合升级之后是否还可以降级？ (2_2023_11_27)
不可以降级


#### 15. 压缩链表包含哪些核心字段？(2_2023_11_27)
```c++
zlbytes 记录压缩列表占用的内存字节数，对压缩列表进行内存重分配的时候使用
zltail 记录压缩列表表尾节点距离压缩列表起始地址有多少字节
zllen 记录压缩列表包含的节点数量，当这个值小于65535时，那么这个值就是节点数量，否则需要便利
entryX 列表节点，数据节点
```

#### 17. 什么是连锁更新？  (2_2023_11_27)
就刚好每个entryX节点都是253字节，然后第一个节点大于255字节了，然后后续节点的previous_entry_length由1字节变成了5字节，从而导致后面所有节点的previous_entry_length都需要修改。
