#### 1.string类型底层数据结构有几种编码？(3_2024_12_15)
redis 2.+版本

1. int：long 类型整数
2. sds embstr: 小于32字节的字符串，申请释放少一次系统调用&充分CPU缓存，重新申请空间redisObject也需要重新申请
4. sds raw: 大于32字节

#### 2. sds核心结构体包含哪些字段？(3_2024_12_15)
```c++
len: 记录buf数组已使用的长度
free: buf数组中未使用的长度
buf[]: 保存字符串
```

#### 3. 为什么使用sds而不使用c自带的字符串？(3_2024_12_15)
1. o(1)时间获取字符串长度
2. 二进制安全
3. 杜绝缓冲区溢出
4. 减少修改字符串带来的内存重分配次数，惰性删除和空间预分配机制
5. 兼容部分c语言函数，已'\0'为结尾

#### 4. hash底层编码包含哪些？(3_2024_12_15)
1. 压缩链表: 512个元素并且每个元素都小于64字节
2. hashtable: 不满足上述条件

#### 6. redis rehash可以缩容嘛？(3_2024_12_15)
可以，容量缩到第一个>= used的2^n次方

#### 7. redis扩缩容的条件？ (3_2024_12_15)
没有执行rdb或者aof，装载因子大于1的时候，执行rdb或者aof，装载因子大于5的时候，执行扩容
装载因子小于0.1的时候只想缩容

装载因子大于1: 翻倍扩容
装载因子小于0.1: 缩容策略

#### 8. redis也是渐进式hash嘛？ (3_2024_12_15)
是的,分散到每一次操作


#### 9. zset的底层编码？ (3_2024_12_15)
1. 压缩链表: 128个元素&每个元素小于64字节
2. 跳表: 不满足压缩链表条件



#### 9. 跳跃表接点包含哪些字段？ (3_2024_12_15)
```c++
level[].span 每层的跨度
level[].forward 前进指针

backward 后退指针
score 分数
obj 对象
```

#### 10. 跳跃表的时间复杂度？(3_2024_12_15)
最好O(logn),最差o(n),平均也是o(logn)的时间复杂度

#### 11.set底层编码？(3_2024_12_15)
1. 整数集合: 数组并且元素个数小于512时整数集合，二次查找去重
2. hash表：不满足整数集合要求


#### 11. 什么时候会使用整数集合？(3_2024_12_15)
整数集合是集合的底层实现之一，当set元素不多，并且都是数字的时候，会使用整数集合

#### 12. 整数集合结构体包含哪些主要字段？(3_2024_12_15)
```c++
encoding 编码
length 长度
content[] 保存元素数组
```

#### 13. 为什么要使用整数结合？ (3_2024_12_15)
节省内存，可以是否二分查找去查找数据在整数集合中是否存在，提升效率


#### 14. 整数集合升级之后是否还可以降级？ (3_2024_12_15)
不可以降级


#### 15. 压缩链表包含哪些核心字段？(3_2024_12_15)
```c++
zlbytes 记录压缩列表占用的内存字节数，对压缩列表进行内存重分配的时候使用
zltail 记录压缩列表表尾节点距离压缩列表起始地址有多少字节
zllen 记录压缩列表包含的节点数量，当这个值小于65535时，那么这个值就是节点数量，否则需要便利
entryX 列表节点，数据节点
```

#### 17. 什么是连锁更新？  (3_2024_12_15)
就刚好每个entryX节点都是253字节，然后第一个节点大于255字节了，然后后续节点的previous_entry_length由1字节变成了5字节，从而导致后面所有节点的previous_entry_length都需要修改。
